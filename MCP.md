# Agent Marketplace — MCP

Protocol basics: MCP_GUIDE.md. MCP servers are a primary *listing type*.

## Strategy
Be the **distribution + monetization layer for MCP servers** (and agents/workflows). Servers generated by #3 publish here; consumers install one-click into ContextOS (#1) where the hub manages them. The marketplace + #3 + #1 form the MCP lifecycle: generate → publish → discover → install → run → monetize.

## MCP servers as listings
- A listing of type `mcp_server` references a hosted server (often from #3) + its tool manifest, permissions, pricing.
- Install = register the server in the consumer's ContextOS (#1) integration hub with scoped permissions.
- Runtime tool calls observed via #4.

## Marketplace-as-MCP-server (optional)
Expose discovery via MCP so an agent can find + install capabilities:
- `search_marketplace(query)` → listings
- `install_listing(listing_id, org)` → install into #1
So an agent can extend its own capabilities from the marketplace (with HITL approval).

## Trust for MCP listings
Security review reuses #3's security lint (auth present, no embedded secrets, injection-aware, safe DB access) + manual review + permission disclosure. Only approved servers installable. See [SECURITY.md](./SECURITY.md), [GUARDRAILS.md](./GUARDRAILS.md).

## Integration
Supply: #3. Install: #1. Observe: #4. The marketplace closes the loop. See [mcp.json](./mcp.json).
