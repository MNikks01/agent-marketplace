# Agent Marketplace — SALES

Two-sided "sales" = recruiting creators (supply) + driving installs (demand), bootstrapped from the platform.

- **First creators:** recruit #3 power users + agencies + #1 users who built useful agents. Pitch: reach the ContextOS user base + earn revenue share + featured placement. Subsidize early (low take-rate, payout guarantees).
- **First consumers:** the #1 user base — promote "install from marketplace" in-product; curate a few must-have listings.
- **First 10 listings / installs:** hand-curated quality catalog; founder-recruited creators; in-#1 promotion.
- **First GMV / $1K take-rate:** a few popular paid listings × usage; subsidize until the flywheel self-sustains.
- **$10K take-rate:** flywheel turning (supply↔demand); enterprise private marketplaces (via #1) add committed revenue.
- **Assets:** creator program + payout terms, curated featured catalog, "install in one click" demo, security-review/trust badges.
- **Objections:** (creators) "will anyone see it?" → built-in #1 distribution + featured + low take-rate; (consumers) "is it safe?" → security review + permissions + sandbox + ratings + observability (#4).
- **Motion:** platform-driven (in-#1) + founder-led creator recruitment. The "sale" is really flywheel ignition — subsidize both sides until network effects take over.
