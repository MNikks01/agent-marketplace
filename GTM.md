# Agent Marketplace — GTM

The hardest GTM in the lab (two-sided, cold-start). The strategy is **sequencing**: launch from the platform's existing audience.

**Positioning:** *"The trusted marketplace for AI agents, MCP servers, and workflows — discover, install one-click into your workflow, and run with confidence."*

## Cold-start strategy (the whole game)
1. **Seed supply first:** convert #3-generated servers + #1 power users' agents into the initial curated catalog (you control day-1 supply).
2. **Seed demand from #1:** the ContextOS user base is the built-in consumer side — promote install-from-marketplace inside #1.
3. **Subsidize the flywheel:** low/zero take-rate early, featured placement, payout guarantees, consumer credits.
4. **Curate hard:** quality + security review from day one (avoid the GPT-Store quality trap).

## Channels
- In-product (ContextOS #1) — primary demand source.
- Creator program / outreach to #3 users + agencies — supply.
- Build-in-public, content ("the agent ecosystem"), PH/HN at public launch.
- MCP community (servers are a core listing type).

## Metrics (flywheel)
Supply: listings, creators, supply growth. Demand: installs, paying consumers, install conversion. Economy: GMV, take-rate revenue, payouts. Balance: supply vs. demand growth. Quality: ratings, takedown rate.

## Why later
Without #1's distribution + #3's supply + #4's trust, this is a ghost town. GTM = patience + sequencing, not a launch blitz. See [VISION.md](./VISION.md), ROADMAP.md.
