# Agent Marketplace — OPEN SOURCE

Open-core (D-007); the marketplace itself is mostly proprietary (it's the network-effect moat), but standards/SDKs are open.

**Open (Apache/MIT):** the **listing manifest spec** (how an agent/MCP-server/workflow declares capabilities, permissions, pricing) — an open standard encourages supply + interop; a **publish SDK/CLI** for creators; security-lint rules (shared with #3).

**Proprietary:** the marketplace platform (catalog, discovery ranking, review pipeline, payouts/economy, curation, private registries, integration with #1/#4). The network effect + curation + trust are the moat — not open-sourceable value.

**Rationale:** open the *manifest standard + publish tooling* (lower the barrier for creators → more supply → flywheel); keep the *marketplace + economy + curation* (where value + defensibility live). A widely-adopted open listing manifest could become the ecosystem standard (strategic), while we run the trusted, monetized hub.

**Licensing:** OSS Apache/MIT for spec/SDK; platform commercial; creator agreement + ToS govern listings. **Fork risk:** a fork can't replicate the network effect, curation, distribution (#1), or trust — those are the moat. Community: manifest + publish-tooling contributions grow supply.
