# Agent Marketplace — SPRINTS

~3–4 months to MVP, but **built LAST (Year 3)** — only after #1/#3/#4 provide distribution, supply, and trust. Sequencing is the whole strategy (avoid cold-start).

## Pre-condition (before any sprint)
ContextOS (#1) has a real user base (distribution); MCP Generator (#3) is producing servers (supply); Agent Monitoring (#4) provides runtime trust. Seed initial catalog from #3 outputs + #1 power users' agents.

## Sprints
- **S0:** Foundation (accounts/roles, reuse platform).
- **S1–2:** Catalog + hybrid discovery + listing detail + ratings.
- **S3–4:** Publishing flow + **one-click install into #1** + permission disclosure. **Exit:** seeded catalog, installable items. (Closed beta with #1 users.)
- **S5–6:** Security review pipeline (reuse #3 lint) + approval gating + takedown.
- **S7–8:** **Stripe Connect** revenue share + metering + creator analytics. **Exit:** creators can earn. **MVP/launch.**

### → MVP COMPLETE (~4 months).

## V2 (later in Y3)
Workflows + bundles + private/org marketplaces + certification + marketplace-as-MCP.
## V3 (Y4+)
Enterprise private registries, license/compliance, partner program, ecosystem governance.

**Cadence:** the metric that matters is the **flywheel** (supply↔demand balance), not feature count. Launch only when both sides are seeded. The marketplace is the ecosystem moat — patience on sequencing beats speed.
